# 05 — Brief

> **What?** The evolution of executive communication across the engagement — from initial framing through to the final pre-production brief.
>
> **Why?** How an idea is communicated determines whether it gets resourced. The brief is not a summary of the architecture — it is the architecture's first user test.
>
> **What breaks?** Technical depth without narrative scaffolding. An executive brief that leads with the architecture rather than the problem loses its audience before the relevant decision is reached.
>
> **What learned?** The most consequential edit across the brief iterations was not architectural — it was a scope boundary. Explicitly declaring what this engagement does not solve (CMS decoupling, Intelligence Vault, historical data migration) created more stakeholder confidence than the technical specification did. Constraints are a trust signal.
